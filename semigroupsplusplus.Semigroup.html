Classsemigroupsplusplus::Semigroup

Class for representing a semigroup consisting of semigroupsplusplus::Elements and defined by a generating set.

Variables

LIMIT_MAX size_t

This variable is used to indicate the maximum possible limit that can be used with semigroupsplusplus::Semigroup::enumerate and semigroupsplusplus::Semigroup::enumerate.
UNDEFINED size_t

This variable is used to indicate that a value is undefined, such as, for example, the position of an element that does not belong to a semigroup.

Constructors

Semigroup ( std::vector<semigroupsplusplus::Element *> * gens )

This is the default constructor for a semigroup generated by gens. The generators gens must all be of the same derived subclass of the semigroupsplusplus::Element base class. Additionally, gens must satisfy the following:

  1. there must be at least one generator;
  2. the generators must have equal degree semigroupsplusplus::Element::degree;

if either of these points is not satisfied, then an asssertion failure will occur.

There can be duplicate generators and although they do not count as distinct elements, they do count as distinct generators. In other words, the generators of the semigroup are precisely (a copy of) gens in the same order they occur in gens.

The generators gens are copied by the constructor, and so it is the responsibility of the caller to delete the argument gens.

gens

the generators of the semigroup

Semigroup ( Semigroup const & copy,
std::vector<semigroupsplusplus::Element *> * coll,
bool report )

This is a constructor for a semigroup generated by the generators of the semigroupsplusplus::Semigroup::Semigroup, semigroupsplusplus::Semigroup::Semigroup and Semigroup copy and the (possibly) additional generators semigroupsplusplus::Semigroup::gens.

The relevant parts of the data structure of copy are copied and then semigroupsplusplus::Semigroup::add_generators is called. Consequently, the semigroup is (partially or fully) enumerated by this constructor.

The same effect can be obtained by copying copy using the copy constructor and then calling semigroupsplusplus::Semigroup::add_generators. However, this constructor avoids copying those parts of the data structure of copy that semigroupsplusplus::Semigroup::add_generators invalidates anyway. If copy has not been enumerated at all, then these two routes for adding more generators are equivalent.

void
add_generators ( std::unordered_set<semigroupsplusplus::Element *, std::hash<semigroupsplusplus::Element *>, std::equal_to<semigroupsplusplus::Element *>> const * coll,
bool report )

This function can be used to add new generators to the existing semigroup in such a way that any previously enumerated data is preserved and not recomputed, or copied. This can be faster than recomputing the semigroup generated by the old generators and the new in coll.

This method is non-const, and changes the semigroup in place, thereby invalidating possibly previously known data about the semigroup, such as the left or right Cayley graphs.

A generator is only added if it is does not belong in the existing data structure (it may belong to the semigroup but just not be known to belong). If coll is empty or only contains elements that are known to belong to the semigroup, then the semigroup is left unchanged.

Otherwise, the semigroup is returned in a state where all of the previously enumerated elements which had been multiplied by all of the old generators, have now been multiplied by the new generators. This means that after this function is called the semigroup might contain many more elements than before (whether it is fully enumerating or not). It can also be the case that the new generators are the only new elements, unlike say, in the case of groups.

The elements the argument coll are copied into the semigroup, and should be deleted by the caller.

coll

the new generators to be added

void
enumerate ( size_t limit,
bool report )

This is the main method of the Semigroup class, and is where the Froidure-Pin Algorithm is implemented. This method is non-const since it involves partially enumerating the semigroup.

If the semigroup is already fully enumerated, or the number of elements previously enumerated exceeds limit, then calling this function does nothing. Otherwise, enumerate and semigroupsplusplus::Semigroup::enumerate attempts to find at least the maximum of limit and the batch size elements of the semigroup. If the semigroup is fully enumerated, then it knows its left and right Cayley graphs, and a minimal factorisation of every element (in terms of its generating set). All of the elements are stored in memory until the object is destroyed.

void
factorisation ( word_t & word,
pos_t pos,
bool report )

This method is non-const since it may involve enumerating the semigroup.

If pos is less than the size of this semigroup, then this method changes its first argument word in-place (first clearing it and then) to contain a minimal factorization of the element in position pos of the semigroup with respect to the generators of the semigroup. This method enumerates the semigroup until at least the pos element is known. If pos is greater than the size of the semigroup, then nothing happens and word is not modified, in particular not cleared.

pos_t
fast_product ( pos_t i,
pos_t j )

This method is const, it asserts that the values i and j are valid, and it either:

  • follows the path in the right or left Cayley graph from i to j, whichever is shorter (semigroupsplusplus::Semigroup::product_by_reduction); or

  • multiplies the elements in postions i and j together;

whichever is better. This is determined by comparing semigroupsplusplus::Element::complexity and the semigroupsplusplus::Semigroup::length_const of i and j.

For example, if the semigroupsplusplus::Element::complexity of the multiplication is linear and the semigroup is a semigroup of transformations of degree 20, and the shortest paths in the left and right Cayley graphs from i to j are of length 100 and 1131, then it better to just multiply the transformation.

void
next_relation ( std::vector<size_t> & relation,
bool report )

This method is non-const since it involves fully enumerating the semigroup.

This method changes relation in-place so that one of the following holds:

  • relation is a vector consisting of a semigroupsplusplus::letter_t and a semigroupsplusplus::letter_t such that semigroupsplusplus::Semigroup::gens()[relation[0]] == semigroupsplusplus::Semigroup::gens()[ relation[1]], i.e. if the semigroup was defined with duplicate generators;

  • relation is a vector consisting of a semigroupsplusplus::Semigroup::pos_t, semigroupsplusplus::letter_t, and semigroupsplusplus::Semigroup::pos_t such that this[relation[0]] * semigroupsplusplus::Semigroup::gens()[relation[1]] == this[relation[2]]

  • relation is empty if there are no more relations.

next_relation is guaranteed to output all relations of length 2 before any relations of length 3. If called repeatedly after semigroupsplusplus::Semigroup::reset_next_relation, and until relation is empty, then the values placed in relation correspond to a length-reducing confluent rewriting system that defines the semigroup.

This can be used in conjunction with semigroupsplusplus::Semigroup::factorisation to obtain a presentation defining the semigroup.

See also semigroupsplusplus::Semigroup::reset_next_relation.

container for the next relation, changed in-place.

void
set_batch_size ( size_t batch_size )

This method is non-const since it changes a class data member, but it does not enumerate the semigroup any further.

The batch size is the number of new elements to be found by any call to semigroupsplusplus::Semigroup::enumerate and semigroupsplusplus::Semigroup::enumerate. A call to semigroupsplusplus::Semigroup::enumerate and semigroupsplusplus::Semigroup::enumerate returns between 0 and approximately the batch size.

The default value of the batch size is 8192.

This is used by, for example, semigroupsplusplus::Semigroup::position so that it is possible to find the position of an element without fully enumerating the semigroup. The value of the batch size is also used to initialise certain data members, if you know a good upper bound for the size of your semigroup, then it is a good idea to call set_batch_size with that upper bound as an argument, this can significantly improve the performance of the semigroupsplusplus::Semigroup::enumerate and semigroupsplusplus::Semigroup::enumerate method.

a new value for the batch size.
