size_t
complexity ( )

This method returns an integer which represents the complexity of multiplying two objects in the same subclass of semigroupsplusplus::Element::Element which have the same semigroupsplusplus::Element::degree. For example, the complexity of multiplying two 3 by 3 matrices over a common semiring is O(3 ^ 3), and 27 is returned by semigroupsplusplus::MatrixOverSemiring::complexity.

The returned value is used in, for example, semigroupsplusplus::Semigroup::fast_product and semigroupsplusplus::Semigroup::nr_idempotents to decide if it is better to multiply elements or follow a path in the Cayley graph.

return

the complexity of multiplying two elements.

size_t
degree ( )

This method returns an integer which represents the size of the element, and is used to determine whether or not two elements are compatible for multiplication. For example, two semigroupsplusplus::Transformations of different degrees cannot be multiplied, or a semigroupsplusplus::Bipartition of degree 10 cannot be an element of a monoid of bipartitions of degree 3.

See the relevant subclass for the particular meaning of the return value of this method for each subclass.

return

the degree of an element.

Element *
really_copy ( size_t increase_deg_by )

This method really copies an semigroupsplusplus::Element::Element. To minimise the amount of copying when semigroupsplusplus::Element::Elements are inserted in unordered_maps and other containers, semigroupsplusplus::Element::Elements behave somewhat like pointers, in that the actual data in an semigroupsplusplus::Element::Element is only copied when this method is called. Otherwise, if an semigroupsplusplus::Element::Element is copied, then its defining data is only stored once.

See also semigroupsplusplus::Element::really_delete.

void
really_delete ( )

This method really deletes an semigroupsplusplus::Element::Element. To minimise the amount of copying when semigroupsplusplus::Element::Elements are inserted in unordered_maps and other containers, semigroupsplusplus::Element::Elements behave somewhat like pointers, in that the actual data in an semigroupsplusplus::Element::Element is only deleted when this method is called. Otherwise, if an semigroupsplusplus::Element::Element is deleted, then its defining data is not deleted, since it may be contained in multiple copies of the element.

See also semigroupsplusplus::Element::really_copy.
